# Raven AI – Golden Scenarios Runbook

This document describes the curated **golden conversation flows** used to validate
Raven AI Agent in the v2.sysmayal.cloud sandbox before any production rollout.

These scenarios exercise:
- Intent resolver
- Multi-agent router (workflow, manufacturing, payments)
- Context manager and agent bus
- Truth hierarchy / anti-hallucination layer

---

## 1. Golden Transcripts Location

Golden transcripts are generated by:

```bash
cd ~/frappe-bench
bench --site v2.sysmayal.cloud execute raven_ai_agent.cli.capture_golden_transcripts.run_and_capture
```

Output files (per run):

- **JSON:** `apps/raven_ai_agent/raven_ai_agent/transcripts/golden_<timestamp>.json`
- **Markdown:** `apps/raven_ai_agent/raven_ai_agent/transcripts/golden_<timestamp>.md`

For the March 22, 2026 baseline, use:

- `raven_ai_agent/raven_ai_agent/transcripts/golden_20260322_014046.json`
- `raven_ai_agent/raven_ai_agent/transcripts/golden_20260322_014046.md`

These should be copied or mirrored into your production runbook repo
(e.g. `infrastructure/runbooks/raven/golden_20260322_014046.*`).

## 2. Scenario A – SO Lifecycle

Entry point (Python):

```python
from raven_ai_agent.cli.so_lifecycle_scenario import run_so_lifecycle_scenario
run_so_lifecycle_scenario(user="runbook_user@yourcompany.com", channel="Raven Golden Runs")
```

High-level flow:

1. `@ai list recent sales orders` – pending SOs dashboard
2. `@ai status SO-00763-LORAND LABORATORIES` – quick SO status
3. `@ai full status SO-00763-LORAND LABORATORIES` – full SO + WOs
4. `@ai show work orders for SO-00763-LORAND LABORATORIES` – active WOs table
5. `@ai mfg status SO-00763-LORAND LABORATORIES` – WO allocation dashboard
6. `@ai payment outstanding` – global outstanding invoices
7. `@ai run SO-00763-LORAND LABORATORIES` – full workflow execution (8-step pipeline)
8. `@ai status SO-00763-LORAND LABORATORIES` – updated pipeline dashboard

**What this proves:**

- Manufacturing and workflow orchestrator can run a full SO pipeline.
- Truth hierarchy + dashboards stay consistent before/after workflow execution.
- Agent can surface blocking issues and partial progress (e.g. 5/8 steps).

## 3. Scenario B – Workflow + MFG + Payments

Entry point (Python):

```python
from raven_ai_agent.cli.so_lifecycle_scenario import run_scenario_2
run_scenario_2(user="runbook_user@yourcompany.com", channel="Raven Golden Runs")
```

Commands:

1. `@ai workflow status SO-00763-LORAND LABORATORIES` – 8-step pipeline dashboard
2. `@ai mfg status SO-00763-LORAND LABORATORIES` – WO allocation (fully allocated)
3. `@ai show work orders for SO-00763-LORAND LABORATORIES` – WO table (4 completed WOs)
4. `@ai full status SO-00763-LORAND LABORATORIES` – SO status + linked WOs
5. `@ai diagnose SO-00763-LORAND LABORATORIES` – diagnosis + next actions
6. `@ai next steps SO-00763-LORAND LABORATORIES` – explicit next-step checklist
7. `@ai payment outstanding customer LORAND LABORATORIES LLC` – per-customer outstanding (ACC-SINV-2026-00071, 107,884.42 USD)
8. `@ai pipeline status SO-00763-LORAND LABORATORIES` – same pipeline view, confirming state

**What this proves:**

- Workflow orchestrator can summarize end-to-end state of a complex SO.
- Manufacturing agent and workflow orchestrator views are consistent.
- Payment agent can answer customer-level outstanding balances tied to a specific SO.

## 4. How to Use in Production Runbooks

For each new environment (staging, then production):

1. Ensure demo data (or anonymized equivalents) for:
   - A representative long-running SO (e.g. SO-00763-LORAND LABORATORIES)
   - At least one outstanding invoice for the same customer.

2. Run the golden transcripts command against that site:
   ```bash
   bench --site <site> execute raven_ai_agent.cli.capture_golden_transcripts.run_and_capture
   ```

3. Compare:
   - That all 16 commands execute without errors.
   - That each step returns concrete, business-meaningful data (no generic help texts).
   - That workflow + manufacturing + payment views are internally consistent.

4. Attach the generated Markdown file to the deployment ticket or change request as evidence of a successful Raven validation run.

5. If any step fails or returns only help text, treat that as a pre-deploy blocker and investigate before enabling Raven for end users.
